# Settings

The **Settings** page (`/settings`) covers instance-wide options that don't
belong to a more specific page:

- **Default template** — applied to any screen that doesn't have its own
  template override.
- **Timezone** — used for scheduling and active-window content.
- **Hide powered-by branding**
- **Hide community links**
- **Hide helping hand** — an onboarding/help widget.
- **Hide demo mode** — hides the Demo Mode page/feature.
- **Welcome headline / welcome text** — shown on a default/landing view.

Telegram and Pretalx settings are instance-level too, but live on their own
dedicated pages — see [Alerting](alerting.md) and [Pretalx](pretalx.md).
